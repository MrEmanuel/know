# Concept

Concepts are optional vocabulary in Know. They describe domain nouns that are shared by many rules, but the system is centered on rules and anchors.

## Concept example

In a hypothetical system, `label` can be a concept. Entities can be labeled by users, and many rules might mention labels: visibility, permissions, copy behavior, and maturity-specific behavior. A rule can link to `concept.label` so `know context <path>` can include a short vocabulary note after the matching rule is found.
